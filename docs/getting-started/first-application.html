First application

The first XIS page should show the complete rhythm: controller, template, run command, and a test. Testing belongs here, not only in a late reference chapter.

Minimal page controller

package example;

import one.xis.Action;
import one.xis.ModelData;
import one.xis.Page;
import one.xis.WelcomePage;

@WelcomePage
@Page("/hello.html")
class HelloPage {

    private int counter;

    @ModelData
    String greeting() {
        return "Hello XIS";
    }

    @ModelData
    int count() {
        return counter;
    }

    @Action
    void increment() {
        counter++;
    }
}

Matching template

<!doctype html>
<html xmlns:xis="https://xis.one/xsd" lang="en">
<body>
  <h1>${greeting}</h1>
  <p>Count: ${count}</p>
  <button xis:action="increment">Increment</button>
</body>
</html>

Add the test immediately

XIS should feel TDD-friendly from the first page. Build a page, assert the rendered state, click the action, and assert the updated state. The examples in this documentation are intended to come from executable sample projects, so the code shown here stays tied to something that actually compiles and runs.

Treat documentation examples as sample code, not as decoration. The closer they stay to executable projects, the less likely the documentation is to drift away from the framework.
