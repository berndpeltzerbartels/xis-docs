Data / SQL repositories

SQL repositories

XIS SQL is explicit. Reading a row creates an object, but changing that object never changes the database by itself. Writes happen through repository operations.

This is a deliberate contrast to JPA-style persistence contexts. XIS favors explicit repository calls and a decoupled service shape. If an application requires an object-level transaction monitor and automatic dirty checking, JPA may be the better persistence model.

No hidden persistence context

XIS SQL maps rows to objects and objects to repository operations. It does not keep loaded entities in a session and it does not inspect changed objects at the end of a transaction. A database write happens when code calls an insert, update, save, delete, or explicitly annotated repository method.

That style is deliberately simple. It fits services and microfrontend applications where the boundary between reading data, changing data, and publishing UI updates should remain visible in the code. 

import one.xis.sql.CrudRepository;
import one.xis.sql.Entity;
import one.xis.sql.Repository;
import one.xis.sql.Save;
import one.xis.sql.Select;

import java.util.List;

@Entity("customers")
record Customer(Long id, String firstName, String lastName) {
}

@Repository
interface CustomerRepository extends CrudRepository<Customer, Long> {

    @Select("select * from customers where last_name = {lastName}")
    List<Customer> findByLastName(String lastName);

    @Save
    Customer save(Customer customer);
}
Customer customer = customerRepository.findById(42L).orElseThrow();
customer = new Customer(customer.id(), "Ada", customer.lastName());

// Nothing has been written yet. The write is explicit:
customerRepository.save(customer);

Schema changes are part of the application

The DDL builder and change-set model give XIS applications a small migration mechanism without turning SQL into an ORM. Change sets are explicit, ordered, and recorded in the database so the application can evolve its schema in a controlled way. 

import one.xis.ddl.Change;
import one.xis.ddl.ChangeSet;
import one.xis.ddl.DDL;

@ChangeSet("customer-schema")
class CustomerSchema {

    @Change("001-create-customers")
    void createCustomers(DDL ddl) {
        var customers = ddl.createTableIfNotExists("customers");
        customers.addColumn("id").bigint().generatedIdentity().primaryKey();
        customers.addColumn("name").varchar(200).notNull();
        customers.addColumn("email").varchar(200).notNull();
    }
}
