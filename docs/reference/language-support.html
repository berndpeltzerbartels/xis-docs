Reference / Language support

Language support

Java is the canonical documentation path. Kotlin and Groovy are supported, but their chapters should be honest about requirements, native-image limitations, and tested behavior.

Java as the main path

The main documentation examples use Java. That keeps the reference path predictable and matches the strongest test and usage focus. 

import one.xis.ModelData;
import one.xis.Page;

@Page("/hello.html")
class HelloPage {
    @ModelData("message")
    String message() {
        return "Hello from Java";
    }
}

Kotlin

Kotlin controllers and form objects are part of the supported surface. Kotlin is also relevant for native images, because Java and Kotlin are the intended native application languages. 

import one.xis.ModelData
import one.xis.Page

@Page("/hello-kotlin.html")
class HelloKotlinPage {
    @ModelData("message")
    fun message(): String = "Hello from Kotlin"
}

Groovy

Groovy is supported on the JVM path and has tests, but it is not the main development path. XIS documents that distinction explicitly so Groovy is neither overstated as the main path nor treated as accidental support. 

import one.xis.ModelData
import one.xis.Page

@Page("/hello-groovy.html")
class HelloGroovyPage {
    @ModelData("message")
    String message() {
        "Hello from Groovy"
    }
}
