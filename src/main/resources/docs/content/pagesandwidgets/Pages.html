Pages

Pages are the entry points of your XIS application. They are controllers annotated with @Page that map to specific URLs and render complete HTML pages in response to HTTP requests.

Defining a Page

A page controller is a Java class annotated with @Page("/path"), where the path parameter specifies the URL at which the page will be accessible. Each page controller has a corresponding HTML template file located in the same package following XIS's convention-based approach. 

@Page("/hello")
public class HelloPage {
    
    @ModelData
    public String greeting() {
        return "Hello World!";
    }
}

The corresponding HelloPage.html template would be placed in the same package and can access the model data provided by the controller: 

<xis:template xmlns:xis="http://www.w3.org/1999/XSL/Transform">
    <h1>${greeting}</h1>
</xis:template>
Page Lifecycle

When a user navigates to a page's URL, XIS processes the request through the following steps:

  • The ControllerService routes the request to the appropriate page controller
  • Controller methods annotated with @ModelData are invoked to provide data
  • The HTML template is rendered with the provided model data
  • The resulting HTML is sent as the HTTP response
Page vs Widget

Unlike widgets, pages are directly accessible via URL and represent complete, standalone views. Pages can embed widgets to compose complex interfaces from reusable components.
