ℹ️ Annotation Changes Coming
Currently, XIS uses prefixed annotations like @XISComponent, @XISService, etc. In a future version, these will be simplified to @Component, @Service, etc. This documentation shows the future naming convention.

XIS Boot provides a lightweight dependency injection and component scanning system. Components are registered and managed by the XIS context, similar to Spring or Micronaut but without the framework overhead.

Component Annotations
@Component

Mark a class as a managed component. XIS will create a single instance and make it available for injection:

@Component
public class UserService {
    public User findById(String id) {
        // Business logic
    }
}
@Service

Specialized @Component for service layer classes:

@Service
public class EmailService {
    public void sendEmail(String to, String subject, String body) {
        // Email sending logic
    }
}
@Repository

Specialized @Component for data access layer classes:

@Repository
public class UserRepository {
    public List<User> findAll() {
        // Database access
    }
}
Dependency Injection

Inject components into pages, widgets, or other components using constructor injection:

@Page("/users.html")
public class UsersPage {
    private final UserService userService;
    
    public UsersPage(UserService userService) {
        this.userService = userService;
    }
    
    @ModelData("users")
    public List<User> getUsers() {
        return userService.findAll();
    }
}
Scopes
@Singleton (Default)

One instance per application (default behavior for @Component):

@Component
@Singleton
public class ConfigService {
    // Single instance shared across the application
}
@RequestScoped

New instance per HTTP request:

@Component
@RequestScoped
public class RequestContext {
    // New instance for each request
}
Component Scanning

XIS Boot automatically scans for components in the same package as your main class and all sub-packages. You can customize the scan path:

public class Application {
    public static void main(String[] args) {
        XisApplication.builder()
            .scanPackage("com.example")          // Main package
            .scanPackage("com.shared.components") // Additional package
            .build()
            .run(args);
    }
}
Manual Component Registration

Register components programmatically:

public class Application {
    public static void main(String[] args) {
        XisApplication.builder()
            .register(MyService.class)
            .registerInstance("config", new AppConfig())
            .build()
            .run(args);
    }
}
Current vs Future Annotations

Mapping of current to future annotation names:

Current Future Purpose
@XISComponent @Component Generic managed component
@XISService @Service Service layer component
@XISRepository @Repository Data access component

Note: Use the current @XIS* prefixed annotations in your code today. The migration to unprefixed annotations will be handled automatically in a future release.
