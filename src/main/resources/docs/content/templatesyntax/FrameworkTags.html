XIS provides a comprehensive set of framework tags for building dynamic templates. Most tags can be written as both elements (<xis:tag>) and attributes (xis:attribute). During processing, attribute syntax gets normalized to tag syntax internally, so both approaches work identically.

Important: Expressions in framework attributes can often be written with or without $​{...}. For example, array="users" and array="$​{users}" both work when the context makes it clear an expression is expected.

Core Control Flow Tags
<xis:foreach>

Iterates over collections to repeat content for each item.
Note: For array and xis:foreach, the expression can also be written without ${...}, e.g. array="users" or xis:foreach="user:users". This is always possible when it is clear that an expression is expected.

Element syntax:

<xis:foreach var="user" array="users">
    <div class="user-card">
        <h3>${user.name}</h3>
        <p>${user.email}</p>
    </div>
</xis:foreach>

Attribute syntax:

<div xis:foreach="user:users">
    <div class="user-card">
        <h3>${user.name}</h3>
        <p>${user.email}</p>
    </div>
</div>

Repeat syntax (alternative):

<div xis:repeat="user:users">
    <div class="user-card">
        <h3>${user.name}</h3>
        <p>${user.email}</p>
    </div>
</div>
<xis:if>

Conditionally renders content based on expressions.
Note: Here too, the expression can be written with or without #&36;{...}, e.g. condition="user.isActive" or xis:if="user.isActive".

Element syntax:

<xis:if condition="user.isActive">
    <div class="status-active">User is active</div>
</xis:if>

Attribute syntax (xis:if):

<div xis:if="user.isActive">
    <div class="status-active">User is active</div>
</div>

Attribute syntax (xis:visible):

<div class="status-active" xis:visible="user.isActive">
    User is active
</div>

DOM Result: The xis:visible version surrounds the element with an if-tag, while xis:if wraps only the content.

<xis:widget-container>

Creates a container for dynamic widget loading.
Note: default-widget is optional, the container can also be initially empty.
container-id (or xis:widget-container as attribute) is the ID that e.g. target-container refers to.
target-container can be used on links, actions, buttons etc. to update a specific widget container. The value must match the container-id of an existing widget container.

Element syntax:

<xis:widget-container container-id="mainContent">
    <!-- initially empty -->
</xis:widget-container>
<xis:widget-container container-id="mainContent" default-widget="DashboardWidget">
    <!-- with default widget -->
</xis:widget-container>

Attribute syntax:

<div xis:widget-container="mainContent"></div>
<div xis:widget-container="mainContent" xis:default-widget="DashboardWidget"></div>

Example for target-container:

<xis:widget-container container-id="main"></xis:widget-container>
<button xis:action="refresh" target-container="main">Refresh Widget</button>
<xis:message>

Displays validation messages for form fields.
Note: message-for can be used as attribute or as tag.

Element syntax:

<input type="text" name="email" />
<xis:message message-for="email"></xis:message>

Attribute syntax:

<div xis:message-for="email"></div>

DOM Result:

<div>
    <xis:message message-for="email"></xis:message>
</div>
<xis:parameter>

Passes parameters to actions and widgets.

<button xis:action="deleteUser">
    <xis:parameter name="userId" value="${user.id}"/>
    <xis:parameter name="confirmRequired" value="true"/>
    Delete User
</button>
Form Elements

XIS provides framework alternatives to standard HTML form elements:

<xis:form>

XIS form element that gets normalized to standard HTML form.

<xis:form binding="user" action="saveUser">
    <xis:input binding="name" type="text"/>
    <xis:input binding="email" type="email"/>
    <xis:submit>Save User</xis:submit>
</xis:form>

DOM Result:

<form xis:binding="user" xis:action="saveUser">
    <input xis:binding="name" type="text"/>
    <input xis:binding="email" type="email"/>
    <input type="submit" value="Save User"/>
</form>
<xis:input>

XIS input element normalized to HTML input.

<xis:input binding="username" type="text" required="true"/>

DOM Result:

<input xis:binding="username" type="text" required="true"/>
<xis:select>

XIS select element normalized to HTML select.

<xis:select binding="country">
    <option value="DE">Germany</option>
    <option value="US">United States</option>
</xis:select>

DOM Result:

<select xis:binding="country">
    <option value="DE">Germany</option>
    <option value="US">United States</option>
</select>
<xis:checkbox>

XIS checkbox element normalized to HTML input with type="checkbox".

<xis:checkbox binding="newsletter" value="true"/>
<label for="newsletter">Subscribe to newsletter</label>

DOM Result:

<input xis:binding="newsletter" type="checkbox" value="true"/>
<label for="newsletter">Subscribe to newsletter</label>
<xis:radio>

XIS radio button element normalized to HTML input with type="radio".

<xis:radio binding="gender" value="male"/>
<label>Male</label>
<xis:radio binding="gender" value="female"/>
<label>Female</label>

DOM Result:

<input xis:binding="gender" type="radio" value="male"/>
<label>Male</label>
<input xis:binding="gender" type="radio" value="female"/>
<label>Female</label>
<xis:button>

XIS button element normalized to HTML button.

<xis:button action="submitForm" type="submit">
    Submit Form
</xis:button>

DOM Result:

<button xis:action="submitForm" type="submit">
    Submit Form
</button>
<xis:submit>

XIS submit button element normalized to HTML submit input.

<xis:submit>Save Changes</xis:submit>

DOM Result:

<submit>Save Changes</submit>
Navigation Elements
<xis:a>

XIS link element that gets normalized to standard HTML anchor. Can navigate to pages or load widgets into containers.
Note: The widget attribute loads a widget - either into the surrounding widget container or into a specific target-container.

Page navigation:

<xis:a page="/user-profile.html" parameters="userId=${user.id}">
    View Profile
</xis:a>

Widget loading:

<xis:a widget="DashboardWidget" target-container="main">
    Show Dashboard
</xis:a>
<!-- Or using attribute syntax: -->
<a xis:widget="DashboardWidget" xis:target-container="main">Show Dashboard</a>

DOM Result:

<a xis:page="/user-profile.html" xis:parameters="userId=${user.id}">
    View Profile
</a>
<a xis:widget="DashboardWidget" xis:target-container="main">
    Show Dashboard
</a>
<xis:action>

XIS action link element for triggering controller actions.

<xis:action action="deleteUser" target-container="userList">
    <xis:parameter name="userId" value="${user.id}"/>
    Delete
</xis:action>

DOM Result:

<a xis:action="deleteUser" xis:target-container="userList">
    Delete
</a>
Special Purpose Tags
<xis:template>

Wrapper tag for widget templates that need multiple root elements. Only required when a widget template doesn't have a single HTML root element. Pages don't need this - they use standard HTML structure.

<!-- Widget with multiple root elements -->
<xis:template xmlns:xis="http://www.w3.org/1999/XSL/Transform">
    <div class="header">Header</div>
    <div class="content">Content</div>
    <div class="footer">Footer</div>
</xis:template>
<xis:raw>

Outputs content without XIS processing. Useful for documentation or when you need to display XIS syntax literally. Has two modes:

HTML mode (default): Inserts content as HTML

<xis:raw>
    <div>This HTML is inserted as-is: $​{not.processed}</div>
</xis:raw>

Text mode: Inserts content as plain text

<xis:raw text="true">
    This is plain text: <div> tags are shown literally $​{not.processed}
</xis:raw>
Complete Example

Tags can be combined and nested for complex scenarios:

<!DOCTYPE html>
<html xmlns:xis="https://xis.one/xsd">
<body>
    <div xis:if="$​{users.size() > 0}">
        <h2>Active Users ($​{users.size()})</h2>
        <div xis:foreach="user:$​{users}">
            <div class="user-card" xis:visible="$​{user.isActive}">
                <h3>$​{user.name}</h3>
                <xis:form binding="user">
                    <xis:input binding="email" type="email"/>
                    <xis:message message-for="email"></xis:message>
                    <xis:button action="updateUser">
                        <xis:parameter name="userId" value="$​{user.id}"/>
                        Update
                    </xis:button>
                </xis:form>
            </div>
        </div>
    </div>
    <div xis:if="$​{users.size() == 0}">
        <p>No users found.</p>
    </div>
</body>
</html>

Key Points:

  • Element syntax (<xis:tag>) and attribute syntax (xis:attribute) are interchangeable
  • During DOM normalization, attributes are converted to tags internally
  • xis:if wraps content, xis:visible wraps the element itself
  • xis:foreach repeats content, xis:repeat repeats the element itself
  • Form elements (<xis:form>, <xis:input>, etc.) normalize to standard HTML with xis: attributes
  • Use <xis:raw> to prevent XIS processing of template syntax
