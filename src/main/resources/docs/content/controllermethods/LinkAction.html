Actions can be triggered by links using the xis:action attribute. This allows you to execute controller methods when users click on links, without requiring buttons or forms.

Basic Link Action

Use xis:action on an <a> tag to trigger a controller method:

@Action
void markAsRead() {
    message.read = true;
    messageService.update(message);
}

Template:

<a xis:action="markAsRead">Mark as read</a>
Link Action with Parameters

Pass parameters to the action method using <xis:parameter> tags:

@Action
void deleteItem(@ActionParameter("itemId") String itemId) {
    itemService.delete(itemId);
}

Template:

<a xis:action="deleteItem">
    <xis:parameter name="itemId" value="${item.id}"/>
    Delete
</a>
Combining Action and Widget Navigation

A link can both execute an action and navigate to a widget. The action executes first, then navigation occurs:

@Action
void selectCategory(@ActionParameter("categoryId") String categoryId) {
    this.selectedCategoryId = categoryId;
}

Template:

<a xis:action="selectCategory" xis:widget="ProductListWidget">
    <xis:parameter name="categoryId" value="${category.id}"/>
    View Products
</a>
Styling Action Links

Action links are regular HTML links and can be styled with CSS classes as needed:

<a xis:action="archive" class="btn btn-secondary">Archive</a>

Note: Use xis:action on links for non-critical actions. For important state changes (like deletions or payments), prefer buttons within forms for better accessibility and to prevent accidental triggers.
