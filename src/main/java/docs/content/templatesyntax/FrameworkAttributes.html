Framework Attributes

XIS framework attributes provide a convenient alternative syntax to framework tags. All attributes are automatically normalized to their corresponding tag syntax during DOM processing. This means you can choose whichever syntax is more readable for your use case - both work identically.

Key Concept: When you write xis:foreach="user:$​{users}" as an attribute, XIS internally transforms it to <xis:foreach var="user" array="$​{users}">. The normalization happens automatically before handlers are created.

Iteration Attributes
xis:foreach

Repeats the content of an element for each item in a collection. The attribute syntax embeds a <xis:foreach> tag as a child of the element.

Attribute Syntax:

<ul xis:foreach="user:$​{users}"> <li>$​{user.name} - $​{user.email}</li> </ul>

Normalized To:

<ul> <xis:foreach var="user" array="$​{users}"> <li>$​{user.name} - $​{user.email}</li> </xis:foreach> </ul>

Rendered HTML:

<ul> <li>John Doe - john@example.com</li> <li>Jane Smith - jane@example.com</li> </ul>
xis:repeat

Repeats the element itself for each item in a collection. The difference to xis:foreach is that xis:repeat wraps the element in a <xis:foreach> parent, repeating the element itself rather than just its content.

Attribute Syntax:

<li class="user-item" xis:repeat="user:$​{users}"> $​{user.name} </li>

Normalized To:

<xis:foreach var="user" array="$​{users}"> <li class="user-item"> $​{user.name} </li> </xis:foreach>

Rendered HTML:

<li class="user-item">John Doe</li> <li class="user-item">Jane Smith</li>
Conditional Attributes
xis:if

Conditionally renders the content of an element. Wraps the element's children in a <xis:if> tag, but the element itself remains.

Attribute Syntax:

<div class="status-container" xis:if="$​{user.isActive}"> <span class="badge">Active</span> </div>

Normalized To:

<div class="status-container"> <xis:if condition="$​{user.isActive}"> <span class="badge">Active</span> </xis:if> </div>

Result: The <div> always renders, but the <span> only appears when the condition is true.

xis:visible

Conditionally renders the element itself. Wraps the entire element in a <xis:if> tag, making both the element and its content conditional.

Attribute Syntax:

<div class="status-badge" xis:visible="$​{user.isActive}"> Active User </div>

Normalized To:

<xis:if condition="$​{user.isActive}"> <div class="status-badge"> Active User </div> </xis:if>

Result: The entire <div> only renders when the condition is true.

Widget Container Attributes
xis:widget-container

Defines a container area for dynamic widget loading. The attribute creates a <xis:widget-container> element as a child.

Attribute Syntax:

<div xis:widget-container="mainContent"></div>

Normalized To:

<div> <xis:widget-container container-id="mainContent"></xis:widget-container> </div>
xis:default-widget

Specifies a default widget to load in a widget container. Must be used together with xis:widget-container.

Attribute Syntax:

<div xis:widget-container="mainContent" xis:default-widget="DashboardWidget"></div>

Normalized To:

<div> <xis:widget-container container-id="mainContent" default-widget="DashboardWidget"> </xis:widget-container> </div>
Form & Validation Attributes
xis:binding

Binds a form or input element to a data model for automatic data binding and validation.

Form Binding:

<form xis:binding="user"> <input xis:binding="name" type="text"/> <input xis:binding="email" type="email"/> </form>
xis:message-for

Displays validation messages for a specific form field. Creates a <xis:message> element as a child.

Attribute Syntax:

<div xis:message-for="email"></div>

Normalized To:

<div> <xis:message message-for="email"></xis:message> </div>
xis:error-class

Applies a CSS class to an element when its bound field has validation errors. Used together with xis:binding.

Example:

<input xis:binding="email" xis:error-class="input-error" type="email"/>

Result: When the email field has validation errors, the class input-error is automatically added to the input element.

Action & Navigation Attributes
xis:action

Triggers a controller action when clicked. Can be used on buttons, links, and other clickable elements.

Example:

<button xis:action="deleteUser">Delete</button> <a href="#" xis:action="viewDetails">View Details</a>
xis:page

Navigates to a specific page URL when clicked.

Example:

<a xis:page="/user-profile.html">View Profile</a>
xis:widget

Loads a specific widget into the nearest widget container when clicked.

Example:

<a xis:widget="UserProfileWidget">View Profile</a>
Understanding Normalization

The normalization process happens automatically during DOM initialization. The DomNormalizer class processes all framework attributes and converts them to their corresponding tag syntax. This ensures that handlers only need to deal with one consistent format.

Key Differences:

  • xis:foreach embeds content in a foreach tag → content repeats
  • xis:repeat wraps element in a foreach tag → element repeats
  • xis:if wraps content in an if tag → element stays, content conditional
  • xis:visible wraps element in an if tag → element and content conditional
Complete Example

Attributes can be combined to create complex, dynamic interfaces:

<div class="user-list" xis:if="$​{users.size() > 0}"> <div class="user-card" xis:repeat="user:$​{users}" xis:visible="$​{user.isActive}"> <h3>$​{user.name}</h3> <p>$​{user.email}</p> <button xis:action="editUser"> <xis:parameter name="userId" value="$​{user.id}"/> Edit </button> </div> </div>

For more details on framework tags and their element syntax, see the Framework Tags documentation.
