XIS-Framework-Attribute können als Attribute oder als Tags verwendet werden. Attribute werden beim Parsen zu Framework-Tags normalisiert. Die wichtigsten Attribute und ihre Anwendung:

xis:foreach

Wiederholt Inhalt für jedes Element einer Collection.

Attribut-Syntax:

<div xis:foreach="user:${users}"> <span>${user.name}</span> </div>

Element-Syntax:

<xis:foreach var="user" array="${users}"> <span>${user.name}</span> </xis:foreach>
xis:repeat

Alternative zu xis:foreach, gleiche Wirkung.

Attribut-Syntax:

<div xis:repeat="user:${users}"> <span>${user.name}</span> </div>
xis:if

Bedingte Sichtbarkeit (umhüllt den Inhalt, nicht das Element selbst).

Attribut-Syntax:

<div xis:if="${user.active}"> <span>Aktiv!</span> </div>

Element-Syntax:

<xis:if condition="${user.active}"> <span>Aktiv!</span> </xis:if>
xis:visible

Bedingte Sichtbarkeit (umhüllt das Element selbst).

Attribut-Syntax:

<div xis:visible="${user.active}"> <span>Aktiv!</span> </div>
xis:widget-container

Definiert einen Bereich für dynamische Widgets.

Attribut-Syntax:

<div xis:widget-container="main"></div>

Element-Syntax:

<xis:widget-container container-id="main" default-widget="MyWidget"></xis:widget-container>
xis:default-widget

Standard-Widget für einen Widget-Container.

Attribut-Syntax (nur mit widget-container):

<div xis:widget-container="main" xis:default-widget="MyWidget"></div>

Element-Syntax:

<xis:widget-container container-id="main" default-widget="MyWidget"></xis:widget-container>
xis:message-for

Bindet eine Fehlermeldung an ein Feld.

Attribut-Syntax:

<input name="email" xis:message-for="email">

Element-Syntax:

<xis:message message-for="email"/>
Hinweis

Weitere Framework-Tags und fortgeschrittene Beispiele findest du in FrameworkTags.
