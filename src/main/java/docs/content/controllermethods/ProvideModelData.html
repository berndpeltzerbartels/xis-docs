How to Provide Data to XIS Templates

XIS supports several ways to provide data to templates. The following table summarizes all options, their typical usage, and how they appear in the template.

Source Description Example Variable in Template
@ModelData Return value of a controller method, available as variable in the template (page/widget scope). ${user}, ${products}
@FormData Form data bound from HTML forms, available as variable after form submission. ${user}, ${profile}
@ActionParameter Parameter passed from template actions (e.g. via <xis:parameter>). ${optionId}, ${value}
@WidgetParameter Parameter passed to widgets via widget links. ${categoryId}, ${sortBy}
@QueryParameter / @PathVariable URL query or path parameters, injected into controller methods. ${userId}, ${productId}
@GlobalVariable Reactive variable shared across all components during a request. ${global.currentUserName}
@ClientStorage Reactive variable stored in browser sessionStorage (persists in tab). ${state.selectedTab}
@LocalStorage Reactive variable stored in browser localStorage (persists long-term). ${localStorage.userTheme}
@RequestScope Data shared between controller methods during a single request. ${user} (if provided by a @ModelData method)
Notes
  • All these sources can be combined as needed in a single template.
  • See the detailed documentation for each annotation for advanced usage and examples.
