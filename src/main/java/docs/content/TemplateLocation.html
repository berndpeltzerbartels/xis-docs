XIS provides flexible options for organizing your HTML templates. By default, templates are placed alongside their corresponding Java controllers, but you can also choose to keep them in the traditional resources folder if you prefer a more conventional project structure.

Default: Co-located Templates

The recommended approach in XIS is to place HTML templates directly next to their controller classes in the same package within src/main/java. The Gradle plugin automatically copies these templates to the resources folder during the build process. This creates a transparent one-to-one relationship between controllers and templates, making it immediately clear which template belongs to which controller. 

src/main/java/
  com/example/
    MyPage.java
    MyPage.html      // Template next to controller
Alternative: Resources Folder

If you prefer to keep templates in the resources folder following traditional Java project conventions, you have two options. You can mirror the package structure as a folder hierarchy in src/main/resources, or you can use the @HtmlFile annotation to specify a custom template location.

When mirroring the package structure: 

src/main/resources/
  com/example/
    MyPage.html      // Mirrors package structure
The @HtmlFile Annotation

The @HtmlFile annotation gives you complete control over template locations. It supports both absolute and relative paths, and works for templates in both src/main/java and src/main/resources: 

@Page("/mypage")
@HtmlFile("templates/custom/MyPage.html")
public class MyPage {
    // Controller code
}

Relative paths are resolved from the package directory of the controller class. This flexibility allows you to organize templates in whatever way best suits your project's needs while maintaining the clear connection between controllers and their views.
