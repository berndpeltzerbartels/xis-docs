The XIS Gradle plugin simplifies the integration of XIS into your Gradle-based projects by automating essential build tasks and providing development tooling. This chapter describes all features provided by the plugin.

Installation

To enable the XIS plugin in your project, add it to your build.gradle plugins block: 

plugins {
    id 'one.xis.plugin' version '${xisVersion}'
}
Dependency Management

The plugin automatically manages version configuration for all XIS dependencies, ensuring consistency across your project. This eliminates the need to specify version numbers for individual XIS modules.

Template File Management

One of the core features is enabling HTML template files to be placed alongside Java source files in the same directory structure. This allows for a transparent one-to-one binding between controllers and their templates, making the relationship between code and markup immediately clear.

For further details on template placement and organization, see the Template Location chapter.

Template Generation Task

The plugin registers a Gradle task under xis/templates that generates simple HTML skeleton templates for all annotated controllers (@Page or @Widget). This task helps bootstrap new controllers by creating the corresponding HTML template files automatically.

Usage: 

./gradlew templates
Note: The task will only create template files that don't exist yet. Existing templates are never overwritten, so you can safely run this task multiple times.

Future enhancements will include generating templates with details from the controller, such as forms with proper field bindings and model data placeholders.

Test Generation Task

The plugin provides a Gradle task under xis/tests that generates integration test files for all @Page controllers. These tests use the IntegrationTestContext and provide a basic skeleton to help you get started with testing.

Usage: 

./gradlew tests

The generated test files are placed in src/test/java in the same package as the controller. Each test includes a basic assertion that the page loads successfully. You can then add more specific assertions to verify the page's behavior and content.

Generated Test Example: 

package quickstart.dashboard;

import one.xis.context.IntegrationTestContext;
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;

import static org.assertj.core.api.Assertions.assertThat;

/**
 * Integration test for Dashboard
 * @generated
 */
class DashboardTest {

    private IntegrationTestContext context;

    @BeforeEach
    void setUp() {
        context = IntegrationTestContext.builder()
            .withSingleton(Dashboard.class)
            .build();
    }

    @Test
    void testPageLoads() {
        var result = context.openPage("/dashboard.html");
        var document = result.getDocument();

        assertThat(document).isNotNull();
        // TODO: Add more assertions
    }
}
Note: The task will only create test files that don't exist yet. Existing tests are never overwritten, so you can safely run this task multiple times. Tests are only generated for @Page controllers, not for widgets. Widgets should be tested through the pages that use them, or by creating dedicated test pages if needed.
Runnable JAR Creation

When using xis-boot, the plugin provides a task under xis/runnableJar that creates a standalone executable JAR file. This JAR includes all dependencies and can be run directly with java -jar.

Usage: 

./gradlew runnableJar

The generated JAR will be named yourapp-version-runnable.jar and located in the build/libs directory. This naming convention makes it immediately clear that the JAR is executable and contains all dependencies, preventing confusion with non-executable JARs.

The runnable JAR can be deployed and executed on any system with a compatible Java runtime: 

java -jar build/libs/yourapp-0.0.1-SNAPSHOT-runnable.jar
Annotation Processing

In future Versions, the plugin registers an annotation processor to handle validation tasks for XIS annotations and conventions. This will provide compile-time checks to ensure your controllers and templates.

Summary

The XIS Gradle plugin provides essential automation for XIS-based projects:

  • Automatic dependency version management
  • Co-location of templates with Java sources
  • Template skeleton generation for controllers
  • Integration test generation for pages
  • Annotation-based compile-time validation (in development)
  • Standalone executable JAR creation for xis-boot applications
