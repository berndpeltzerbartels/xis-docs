Let's create your first XIS page with a simple Hello World example. This demonstrates the core concepts of XIS in just a few lines of code.

The Controller

Create a Java class annotated with @Page to define the URL where your page will be accessible. Use the @ModelData annotation on methods to provide data to the template: 

package com.example;

import one.xis.Page;
import one.xis.ModelData;
import one.xis.WelcomePage;

@Page("/hello")
@WelcomePage
public class HelloWorldPage {
    
    @ModelData
    public String greeting() {
        return "Hello World!";
    }
    
    @ModelData
    public String message() {
        return "Welcome to XIS - building web apps the simple way.";
    }
}
The Template

Create an HTML file named HelloWorldPage.html in the same package as your controller. The template uses standard HTML with expression language $​{...} to access the model data: 

<!DOCTYPE html>
<html>
<head>
    <title>Hello World</title>
</head>
<body>
    <h1>$​{greeting}</h1>
    <p>$​{message}</p>
</body>
</html>
That's It!

Start your application and navigate to http://localhost:8080/hello.html. XIS automatically wires everything together - the controller provides the data, the template renders it, and you have a working web page. No configuration files, no complex setup, just plain Java and HTML working together.

The @ModelData annotation makes the method's return value available in the template using the method name as the key. For more details on controller methods and annotations, see the dedicated chapters later in this documentation.

@WelcomePage marks this page as the default landing page of your application. When users visit the root URL (/) or any unknown URL, they will be redirected to this page automatically.
