The Problem

Modern web development often requires massive amounts of boilerplate code. Consider adding a simple "Product" entity to your application:

Traditional Stack (React + Spring)
  • REST Controller (~50 lines)
  • Service Layer (~40 lines)
  • DTO + Assembler (~60 lines)
  • React Component (~80 lines)
  • Redux Actions (~30 lines)
  • Redux Reducer (~50 lines)
  • API Client (~40 lines)
  • TypeScript Types (~20 lines)

Total: ~370 lines of code

With XIS
  • @Page class (~15 lines)
  • HTML Template (~25 lines)

Total: ~40 lines of code

90% less boilerplate

The Root Cause

The traditional separation between frontend and backend creates constant friction:

  • APIs must be discussed, designed, documented, and versioned
  • Changes require coordination between teams and propagate through all layers
  • Data transformations happen multiple times (Entity → DTO → JSON → Type → State)
  • State management adds complexity even for simple CRUD operations
  • Every layer requires its own tests—unit tests, integration tests, E2E tests
  • A single feature change touches 6-8 files plus their corresponding test files

The XIS Approach

${frameworkName} eliminates the artificial frontend-backend boundary for feature development. Instead of splitting your application horizontally (frontend team vs. backend team), XIS enables vertical feature slices:

  • Single Page Application with automatic UI updates
  • Annotation-driven development for clean, declarative code
  • Automatic state synchronization without Redux or MobX
  • Easy test-driven development with minimal mocking
  • Type-safe data binding between HTML and Java
  • Component-based architecture with true microfrontend capabilities
  • Fast and lightweight with minimal dependencies

Write your UI in HTML templates, add behavior with Java annotations, and let XIS handle the plumbing. No REST API design. No state management boilerplate. No frontend-backend coordination overhead.

Focus on your features, not on infrastructure.
