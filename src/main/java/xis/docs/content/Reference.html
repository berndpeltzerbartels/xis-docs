Navigation Attributes

xis:page

Usage: Navigate to a different page

Supported elements: <a>, <button>

Example:

<a xis:page="/user-profile.html">View Profile</a>
<button xis:page="/dashboard.html">Go to Dashboard</button>
xis:widget

Usage: Load a widget into a widget container

Supported elements: <a>, <button>

Additional attributes:

  • xis:target-container - Optional. Specifies the container ID where the widget should be loaded. If not specified, loads into the parent widget container.

Example:

<a xis:widget="DashboardWidget">Dashboard</a>
<button xis:widget="UserListWidget" xis:target-container="main">Show Users</button>
xis:target-container

Usage: Specifies which widget container should be updated

Used with: xis:widget, xis:action

Supported elements: <a>, <button>

Example:

<xis:widget-container container-id="main"></xis:widget-container>
<button xis:widget="SettingsWidget" xis:target-container="main">Settings</button>

Reactive Attributes

xis:storage-binding

Usage: Binds DOM content to client-side storage for automatic reactive updates

Supported elements: Any element containing expressions that reference storage

Allowed values:

  • "localStorage" - Binds to browser localStorage (persistent across sessions)
  • "sessionStorage" - Binds to browser sessionStorage (tab-scoped, survives reload)
  • "clientStorage" - Binds to JavaScript field storage (short-lived)

Behavior: When the specified storage changes, all expressions within the bound element are re-evaluated and the DOM is updated automatically.

Example:

<!-- Simple counter -->
<span xis:storage-binding="localStorage">
    Items: $​{count(localStorage.cart.items)}
</span>

<!-- Complex aggregation -->
<div xis:storage-binding="localStorage">
    <p>Total items: $​{count(localStorage.shoppingCard.products)}</p>
    <p>Total price: €$​{sum(flatMap(localStorage.shoppingCard,'products.price'))}</p>
</div>

<!-- Session-scoped binding -->
<span xis:storage-binding="sessionStorage">
    Welcome, $​{sessionStorage.user.name}!
</span>

Use with: Works seamlessly with @LocalStorage, @SessionStorage, and @ClientStorage parameter annotations that automatically update storage after action execution.
