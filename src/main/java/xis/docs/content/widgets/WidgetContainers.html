Widget containers define regions on a page where widgets can be dynamically loaded and replaced. They provide the foundation for building modular, component-based UIs.

Basic Widget Container

Use the <xis:widget-container> tag with a container-id to create a widget placeholder. The container ID uniquely identifies the container within the page.

<xis:widget-container container-id="main">
</xis:widget-container>
Default Widget

Specify a default widget to load when the page opens using the default-widget attribute:

<xis:widget-container container-id="main" default-widget="DashboardWidget">
</xis:widget-container>

Note: Older API versions use xis:default-widget instead of default-widget. Both formats are supported.

Passing Parameters: For details on how to pass parameters to widgets via containers, see the Widget Parameters chapter.

Container ID with EL Expressions

Container IDs can contain EL expressions for dynamic container identification:

<xis:widget-container container-id="section-${sectionId}" 
                     default-widget="SectionWidget">
    <xis:parameter name="sectionId" value="${sectionId}"/>
</xis:widget-container>
Multiple Containers on Same Page

A page can have multiple widget containers, each with its own ID:

<div class="layout">
    <aside>
        <xis:widget-container container-id="sidebar" default-widget="NavigationWidget">
        </xis:widget-container>
    </aside>
    <main>
        <xis:widget-container container-id="main" default-widget="DashboardWidget">
        </xis:widget-container>
    </main>
</div>
Scroll to Top on Widget Navigation

Use the scroll-to-top="true" attribute to automatically scroll to the top of the page when navigating to a different widget. This improves user experience by ensuring new content is visible:

<xis:widget-container container-id="main" 
                     default-widget="ArticleWidget" 
                     scroll-to-top="true">
</xis:widget-container>

Behavior:

  • Scrolls to top (0,0) when navigating to a different widget via user action
  • Does not scroll when the page initially loads with the default widget
  • Does not scroll when the same widget is reloaded

Note: The attribute can also be written as xis:scroll-to-top="true" when using attribute syntax: <div xis:widget-container="main" xis:scroll-to-top="true"></div>

Target Container in Actions

Actions can update specific containers using targetContainer. This is covered in detail in the Widget Navigation chapter.
