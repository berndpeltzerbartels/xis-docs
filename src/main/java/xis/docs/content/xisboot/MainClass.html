The main class is the entry point of your XIS Boot application. It bootstraps the XIS runtime and starts the embedded web server.

Basic Main Class

Create a simple main class that starts XIS Boot:

package com.example.myapp;

import one.xis.boot.XisApplication;

public class Application {
    public static void main(String[] args) {
        XisApplication.run(Application.class, args);
    }
}

The XisApplication.run() method:

  • Scans for @Page and @Widget annotated classes in the same package
  • Registers all controllers and components
  • Starts the embedded web server (default port 8080)
  • Processes command-line arguments (e.g., --port=9090)
Configuration

You can customize the application startup:

public class Application {
    public static void main(String[] args) {
        XisApplication.builder()
            .port(9090)
            .contextPath("/myapp")
            .scanPackage("com.example")
            .build()
            .run(args);
    }
}
Application Structure

Organize your application with a clear package structure:

com.example.myapp/
├── Application.java         # Main class
├── pages/
│   ├── HomePage.java
│   └── HomePage.html
└── widgets/
    ├── HeaderWidget.java
    └── HeaderWidget.html
Command Line Arguments

XIS Boot supports common command-line arguments:

  • --port=8080 - Server port (default: 8080)
  • --context-path=/app - Application context path
  • --dev-mode=true - Enable development mode with hot reload

Example:

java -jar myapp.jar --port=9090 --dev-mode=true
