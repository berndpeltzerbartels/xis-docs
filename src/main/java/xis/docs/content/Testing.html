Testing is a core part of XIS development. The framework provides integration testing utilities that allow you to test controllers, pages, and widgets without running a full web server.

Integration Testing

Use IntegrationTestContext to create isolated test environments for your controllers. This enables fast, reliable tests for page and widget logic, including data binding and action handling. 

var context = IntegrationTestContext.builder()
    .withSingleton(MyPage.class)
    .build();
var result = context.openPage(MyPage.class);
assertThat(result.getDocument().getElementById("test").getInnerText())
    .isEqualTo("expected");
Widget Testing

Widgets can be tested in isolation by opening them in the test context and asserting on their rendered output and data. 

var widgetResult = context.openWidget(MyWidget.class);
assertThat(widgetResult.getDocument().getElementById("widgetTitle").getInnerText())
    .isEqualTo("Widget Title");
Browser Testing

For end-to-end browser tests, XIS supports Selenide-based tests. These allow you to automate real browser interactions and verify the complete user experience. 

// Example Selenide test
open("/mypage.html");
$("#myButton").click();
$("#result").shouldHave(text("Success!"));
Best Practices
  • Test both controller logic and rendered output
  • Use integration tests for fast feedback
  • Use browser tests for full-stack verification
  • Mock external dependencies where possible
