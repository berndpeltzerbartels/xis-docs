Prerequisites
  • Java 17 or higher
  • Gradle 8.0 or higher
Framework Options

XIS can be used with different web frameworks. You need to choose one of the following options:

  • Spring Boot - Use XIS with the popular Spring Boot framework (recommended for existing Spring projects)
  • XIS Boot - A lightweight standalone alternative to Spring Boot, specifically designed for XIS
  • Micronaut - Integration is under consideration for future releases
Option 1: Using XIS with Spring Boot

When using Spring Boot, you need both the Spring Boot plugin and the XIS plugin. The Spring Boot dependency management automatically handles version management for Spring dependencies. Maven support is planned for future releases. 

plugins {
    id 'java'
    id 'org.springframework.boot' version '3.3.0'
    id 'io.spring.dependency-management' version '1.1.5'
    id 'one.xis.plugin' version '${xisVersion}'
}

group = 'com.example'
version = '0.0.1-SNAPSHOT'

repositories {
    mavenCentral()
}

dependencies {
    // XIS libraries
    implementation 'one.xis:xis-spring'
    testImplementation 'one.xis:xis-test'

    // Lombok (optional, but recommended)
    compileOnly 'org.projectlombok:lombok:1.18.30'
    annotationProcessor 'org.projectlombok:lombok:1.18.30'

    // Spring Boot (versions managed by Spring Boot plugin)
    implementation 'org.springframework.boot:spring-boot-starter-web'

    // JUnit 5 for testing (versions managed by Spring Boot plugin)
    testImplementation 'org.springframework.boot:spring-boot-starter-test'
}

test {
    useJUnitPlatform()
}

The io.spring.dependency-management plugin provides automatic version management for all Spring dependencies, so you don't need to specify versions for Spring Boot libraries.

Option 2: Using XIS Boot (Standalone Alternative)

XIS Boot is a lightweight alternative to Spring Boot that provides the same functionality without Spring dependencies. It has its own dependency injection context that is intentionally designed to be similar to Spring. This option is ideal if you want a minimal footprint and don't need Spring-specific features. 

plugins {
    id 'java'
    id 'one.xis.plugin' version '${xisVersion}'
}

group = 'com.example'
version = '0.0.1-SNAPSHOT'

repositories {
    mavenCentral()
}

dependencies {
    // XIS Boot - replaces Spring Boot
    implementation 'one.xis:xis-boot'
    testImplementation 'one.xis:xis-test'

    // Lombok (optional, but recommended)
    compileOnly 'org.projectlombok:lombok:1.18.30'
    annotationProcessor 'org.projectlombok:lombok:1.18.30'

    // JUnit 5 for testing
    testImplementation 'org.junit.jupiter:junit-jupiter-api:5.10.0'
    testRuntimeOnly 'org.junit.jupiter:junit-jupiter-engine:5.10.0'
}

test {
    useJUnitPlatform()
}
Important: Do not forget to add the XIS plugin one.xis.plugin to your build.gradle file.
