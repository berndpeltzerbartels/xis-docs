Pages and Widgets in XIS cannot include JavaScript files using standard HTML <script> tags directly in templates. This is because XIS is a Single Page Application (SPA) where content is dynamically inserted into the DOM at runtime. Browsers do not execute <script> tags that are added to the DOM after the initial page load - this is an important browser security feature.

To include custom JavaScript in your XIS application, use the META-INF extension mechanism. This is useful for:

  • Framework extensions and global utilities
  • Third-party library integration
  • Application-wide event handlers
  • Custom Expression Language functions

Create a file at src/main/resources/META-INF/xis/js/extensions that lists your JavaScript files (one per line): 

js/global-utilities.js
js/analytics-tracker.js
js/custom-el-functions.js

Place the referenced JavaScript files in src/main/resources/. These files will be automatically bundled and loaded with the XIS framework at application startup.

CSS and Other Resources

Unlike JavaScript, CSS files and other resources can be included directly in templates using standard HTML tags, as they do not have the same browser execution restrictions: 

<link rel="stylesheet" href="/css/custom-styles.css">
<img src="/images/logo.png" alt="Logo">
