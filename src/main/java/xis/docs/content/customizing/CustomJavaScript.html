Pages and Widgets in XIS cannot include JavaScript files using standard HTML <script> tags directly in templates. This is because XIS is a Single Page Application (SPA) where content is dynamically inserted into the DOM at runtime. Browsers do not execute <script> tags that are added to the DOM after the initial page load - this is an important browser security feature.

Convention-Based JavaScript Inclusion

XIS uses a convention-based approach where you place a .js file with the same name as your controller in the same package. These JavaScript files are automatically discovered, bundled, and included in your application. 

// File structure
src/main/java/
    com/example/
        UserPage.java           // Page controller
        UserPage.html           // HTML template
        UserPage.js             // JavaScript (automatically included)

The JavaScript file will be automatically loaded when the page or widget is rendered. You can access DOM elements and add event listeners as usual: 

// UserPage.js
document.addEventListener('DOMContentLoaded', () => {
    const button = document.getElementById('myButton');
    button.addEventListener('click', () => {
        console.log('Button clicked!');
    });
});
Framework Extensions via META-INF

For global JavaScript that should be available across your entire application (not tied to a specific page or widget), you can use the META-INF extension mechanism. This is particularly useful for:

  • Framework extensions and global utilities
  • Third-party library integration
  • Application-wide event handlers

Create a file at src/main/resources/META-INF/xis/js/extensions that lists your JavaScript files (one per line): 

js/global-utilities.js
js/analytics-tracker.js

Place the referenced JavaScript files in src/main/resources/. These files will be automatically bundled and loaded with the XIS framework.

CSS and Other Resources

Unlike JavaScript, CSS files and other resources can be included directly in templates using standard HTML tags, as they do not have the same browser execution restrictions: 

<link rel="stylesheet" href="/css/custom-styles.css">
<img src="/images/logo.png" alt="Logo">
