XIS provides multiple ways to create links between pages. Let's explore them by adding links to your Dashboard page.

Edit the Dashboard template

Open Dashboard.html and update it with the following content: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8"/>
    <title>Dashboard</title>
</head>
<body>
<h1>Dashboard</h1>
<h2>XIS Examples</h2>
<p>
    <a href="/hello-world.html">Hello World Example (1)</a><br/>
    <a xis:page="/hello-world.html">Hello World Example (2)</a><br/>
    <xis:a page="/hello-world.html">Hello World Example (3)</xis:a>
    <br/>
</p>
</body>
</html>
Restart and test

Restart your application and open the Dashboard in your browser. All three links work!

Understanding the link types

Options 2 and 3 are just different syntax for the same thing - both are XIS-specific page links:

  • <a xis:page="/hello-world.html"> - XIS attribute syntax
  • <xis:a page="/hello-world.html"> - XIS element syntax

These XIS links use the DOM API to replace content without a full page reload, providing a smoother user experience.

The standard HTML link (<a href="...">) also works in modern browsers with the same behavior. However, in older browsers, it will trigger a full page reload.

Recommended approach

We'll use the attribute syntax (xis:page) going forward. Update your Dashboard to keep only this version: 

<p>
    <a xis:page="/hello-world.html">Hello World Example</a>
</p>
