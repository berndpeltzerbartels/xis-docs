Let's create a dashboard page as the entry point for your application using the @WelcomePage annotation.

Create the Dashboard page

Create a new package dashboard inside quickstart and add a file Dashboard.java: 

package quickstart.dashboard;

import one.xis.Page;
import one.xis.WelcomePage;

@Page("/index.html")
@WelcomePage
class Dashboard {
}
What is @WelcomePage?

The @WelcomePage annotation marks a page as the default entry point of your application. When a user navigates to the root URL (http://localhost:8080/), they will be automatically redirected to this page.

Key points:

  • Only one page in your application should have the @WelcomePage annotation
  • It eliminates the need for manual configuration of the default landing page
  • Users can access it via the root URL or the page's specific path (/index.html)
  • It also serves as a fallback for non-existent URLs (e.g., /bla.html) - acting like a 404 error page

With this setup, visiting http://localhost:8080/ will show your Dashboard page.

Create the HTML template

Generate the template for your Dashboard page:

./gradlew templates

Then open the generated Dashboard.html file and add a simple heading: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8"/>
    <title>Dashboard</title>
</head>
<body>
<h1>Dashboard</h1>
</body>
</html>
Test in the browser

Restart your application:

./gradlew bootRun

Now open your browser and navigate to:

http://localhost:8080/

You should see the Dashboard page. Notice that you don't need to specify /index.html - the root URL automatically redirects to your welcome page.
