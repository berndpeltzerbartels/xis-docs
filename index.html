Documentation / Overview

Server-driven SPA development for JVM teams

XIS lets Java, Kotlin, and Groovy developers build interactive SPA-style applications from backend controllers and plain HTML templates. The programming model feels close to classic request/response development, while the browser is updated by the XIS JavaScript runtime without handwritten frontend application plumbing.

Core idea. A feature should be understandable as a vertical slice: controller, template, tests, and optional persistence live close to each other. XIS supplies transport, synchronization, validation feedback, navigation, and partial refreshes. 
import one.xis.ModelData;
import one.xis.Page;

@Page("/hello.html")
class HelloPage {
    @ModelData("message")
    String message() {
        return "Hello from XIS";
    }
}
<h1>${message}</h1>

Who XIS is for

XIS is aimed at backend-oriented JVM teams that still want a modern browser experience. You should be able to build pages, forms, validation feedback, navigation, refreshes, and modal interactions without first creating a separate REST contract, DTO layer, frontend client, and frontend state model.

That does not mean JavaScript is forbidden. It means JavaScript is no longer required for routine application plumbing. When a feature genuinely needs browser-side behavior, XIS offers extension points for custom JavaScript and custom expression-language functions.
